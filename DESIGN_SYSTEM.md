# DESIGN_SYSTEM.md — ITRobotCRM

Маніфест дизайн-системи для CRM. Ці правила — обов'язкові при створенні та модифікації будь-якого UI у проєкті. Мета: сучасний, світлий, професійний інтерфейс із високою читабельністю та передбачуваною структурою.

---

## 1. Філософія

- **Мінімалізм.** Жодних декоративних елементів "для краси". Кожен елемент має функцію.
- **Багато вільного простору (white space).** Простір — не порожнеча, а інструмент ієрархії. Не економити на padding та margin.
- **Світло та чистота.** Інтерфейс світлий, з м'якими сірими фонами для панелей і чисто білими картками поверх них.
- **Передбачуваність.** Однакові сутності виглядають однаково. Користувач має впізнавати компоненти з першого погляду.
- **Контент — головний.** UI не перетягує увагу на себе. Заголовки, дані, кнопки — у фокусі; рамки, тіні, розділювачі — ледь помітні.

## 3. Типографіка

### Шрифт
Використовувати **системний sans-serif стек** — без підключення Google Fonts:

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
```

Для цифр у таблицях/звітах додавати:
```css
font-variant-numeric: tabular-nums;
```

### Шкала розмірів
| Роль | Розмір | Line-height | Weight |
|---|---|---|---|
| Display (великий заголовок сторінки) | 28px / 1.75rem | 36px | 600 |
| H1 (заголовок розділу) | 24px / 1.5rem | 32px | 600 |
| H2 (заголовок картки) | 20px / 1.25rem | 28px | 600 |
| H3 (підзаголовок) | 16px / 1rem | 24px | 600 |
| Body | 14px / 0.875rem | 20px | 400 |
| Small (підписи, метадані) | 13px | 18px | 400 |
| Caption (лейбли, крихти) | 12px | 16px | 500 |

### Правила
- **Контраст між заголовком і текстом:** заголовок — `weight 600` + `--text-primary`; опис — `weight 400` + `--text-secondary`.
- **Максимальна ширина абзацу** — 70ch для довгого тексту.
- **Ніякого `text-align: justify`** — тільки `left`.
- **Ніколи не використовувати `font-weight: 400` для заголовків** і `600+` для body-тексту в таблицях.
- Заголовки модалок — H2, заголовки секцій у модалці — H3.

## 4. Сітка відступів — правило 4px

**Усі відступи (margin, padding, gap) мають ділитися на 4.** Це базовий закон системи.

### Дозволені значення
`0, 4, 8, 12, 16, 20, 24, 32, 40, 48, 64` px

### Типові призначення
| Значення | Де використовувати |
|---|---|
| `4px` | Відступ між іконкою та текстом всередині кнопки/бейджа |
| `8px` | Компактні gap'и, padding у малих бейджах, відступ між лейблом і полем |
| `12px` | Padding у полях вводу, gap між іконкою і текстом пункту меню |
| `16px` | Padding усередині карток (малі), gap між елементами форми |
| `24px` | Padding усередині карток (стандартні), відступ між секціями форми |
| `32px` | Відступ між блоками сторінки, padding у великих модалках |
| `48px` / `64px` | Відступ між великими розділами сторінки |

### Заборонено
- ❌ `5px`, `10px`, `15px`, `18px`, `22px` та будь-які інші значення не кратні 4.
- ❌ Довільні `margin: 13px 17px`.
- ❌ Використовувати `px` там, де в компоненті вже використовується токен/змінна — брати з існуючої системи.

### CSS-токени для відступів
```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
--space-8: 32px;
--space-10: 40px;
--space-12: 48px;
--space-16: 64px;
```

## 5. Скруглення кутів (Border Radius)

Тільки два основних значення:

| Токен | Значення | Де використовувати |
|---|---|---|
| `--radius-sm` | `8px` | Кнопки, поля вводу, бейджі, малі елементи |
| `--radius-md` | `12px` | Картки, модальні вікна, секції, великі блоки |
| `--radius-full` | `9999px` | Аватари, круглі іконки-кнопки, чіпи (за потреби) |

- ❌ Не використовувати `4px`, `6px`, `10px`, `16px`, `20px` радіус.
- ✅ У межах однієї картки всі вкладені елементи використовують `--radius-sm` (8px).

---

## 6. Тіні (Shadows)

Тіні — м'які, майже непомітні. Не драматичні.

```css
--shadow-xs: 0 1px 2px rgba(15, 23, 42, 0.04);
--shadow-sm: 0 1px 3px rgba(15, 23, 42, 0.06), 0 1px 2px rgba(15, 23, 42, 0.04);
--shadow-md: 0 4px 12px rgba(15, 23, 42, 0.08);
--shadow-lg: 0 12px 32px rgba(15, 23, 42, 0.12);
```

| Токен | Де |
|---|---|
| `--shadow-xs` | Статичні картки на світлому фоні |
| `--shadow-sm` | Dropdown, tooltip |
| `--shadow-md` | Popover, draggable-модалка в стані drag |
| `--shadow-lg` | Модальні вікна |

**Замість важких тіней** — використовувати `1px` border з `--border-subtle`. Картки часто краще показувати бордером, а не тінню.

## 7. Іконки

### Бібліотека — **Lucide** (`lucide-react`)
- Тонкі, професійні, однакова сітка.
- **Ніколи не змішувати** Lucide з іншими іконпаками (Heroicons, Feather, Font Awesome).
- ❌ Емодзі в UI заборонено. Навіть як заглушки.

### Розміри
| Контекст | Розмір | stroke-width |
|---|---|---|
| Inline в тексті | `14px` | `1.75` |
| Кнопки, поля | `16px` | `1.75` |
| Sidebar, навігація | `18px` | `1.75` |
| Заголовки карток, порожні стани | `20–24px` | `1.75` |
| Великі feature-іконки | `32–48px` | `1.5` |

### Колір
- За замовчуванням — `currentColor` (іконка наслідує колір тексту).
- Статусні іконки беруть семантичний колір (`--success`, `--danger` тощо).
- Декоративні іконки у вторинному контексті — `--text-muted`.

## 8. Компоненти — базові правила

### Кнопки
- **Висота:** `36px` (стандарт), `32px` (компакт), `40px` (великі CTA).
- **Padding:** по горизонталі `16px` (стандарт), `12px` (компакт).
- **Радіус:** `8px`.
- **Типи:**
  - `primary` — суцільна заливка `--accent-primary`, білий текст.
  - `secondary` — білий фон, бордер `--border-default`, текст `--text-primary`.
  - `ghost` — прозорий фон, ховер `--bg-hover`.
  - `danger` — для деструктивних дій (`--danger` або outline-варіант).
- **Іконка + текст:** gap `8px`, іконка `16px`.
- **Focus:** видимий ring — `0 0 0 3px rgba(59, 130, 246, 0.25)`.

### Поля вводу
- Висота `40px`, padding `12px`, радіус `8px`.
- Бордер `--border-default`; focus — бордер `--accent-primary` + ring.
- Лейбл — 13px, `--text-secondary`, `margin-bottom: 8px`.
- Помилка — текст `--danger`, 13px, `margin-top: 4px`.

### Картки
- Фон `--bg-card`, бордер `1px solid --border-subtle`, радіус `12px`.
- Padding внутрішній `24px` (стандартно).
- Заголовок картки — H2 або H3 + опис (secondary).

### Таблиці
- Border-collapse; роздільники — `1px solid --border-subtle` знизу у рядка.
- Header: `--text-secondary`, 13px, `weight 500`, `text-transform: uppercase`, `letter-spacing: 0.04em`.
- Рядки: висота `48px`, padding по горизонталі `16px`.
- Ховер — фон `--bg-hover`.
- Виділений рядок — фон `--accent-primary-soft`.
- **Числові колонки** — `text-align: right` + `tabular-nums`.

### Бейджі / Chips
- Висота `22–24px`, padding `2px 8px`, радіус `8px` (або `full` для round).
- Розмір тексту `12px`, weight `500`.
- Фон — `*-soft` токен, текст — відповідний насичений токен.

### Модалки
- Використовувати існуючий `DraggableModal`.
- Фон `--bg-card`, радіус `12px`, тінь `--shadow-lg`.
- Заголовок — H2, padding `24px 24px 16px`; контент `24px`; футер з кнопками `16px 24px 24px`, вирівнювання праворуч, gap `8px`.

### Sidebar / Навігація
- Фон `--bg-panel`, ширина desktop ~240px.
- Пункт: висота `40px`, padding `8px 12px`, радіус `8px`, gap `12px` між іконкою (18px) та текстом.
- Активний пункт — фон `--accent-primary-soft`, текст `--accent-primary`, weight `500`.
- Hover — фон `--bg-hover`.

## 9. Стани (States)

Для кожного інтерактивного елемента передбачати:
1. **Default**
2. **Hover** — легка зміна фону (на один рівень темніше) або border.
3. **Focus-visible** — ring із `--accent-primary` @ 25% opacity. Обов'язково для клавіатурної навігації.
4. **Active / Pressed** — трохи темніший стан.
5. **Disabled** — `opacity: 0.5`, `cursor: not-allowed`, без ховера.
6. **Loading** — спінер із Lucide `Loader2` + `animation: spin`.

---

## 10. Анімації

- **Duration:** `150ms` (мікро), `200ms` (стандарт), `300ms` (модалки, великі переходи).
- **Easing:** `cubic-bezier(0.4, 0, 0.2, 1)` (стандарт).
- Анімувати тільки: `opacity`, `transform`, `background-color`, `border-color`, `box-shadow`.
- ❌ Ніколи не анімувати `width/height/top/left` — тільки `transform`.
- Поважати `prefers-reduced-motion: reduce` — анімації вимкнути.

---

## 11. Сітка та layout

- **Max-width контенту:** 1280px для адмінських сторінок, центрування.
- **Gutter сторінки:** `24px` (mobile) → `32px` (desktop).
- **Grid-gap між картками:** `16px` (щільні списки) → `24px` (стандарт).
- Уникати фіксованих висот там, де можливий overflow. Довгі списки — скрол усередині картки, а не сторінки.

---

## 12. Accessibility (обов'язково)

- Контраст тексту до фону — мінімум **WCAG AA (4.5:1)** для body, **3:1** для великого тексту.
- Усі інтерактивні елементи мають `focus-visible` стиль.
- Іконки без текстової підписки — обов'язково `aria-label`.
- Мінімальна клікабельна область — `32×32px` (краще `40×40px`).
- Форми — кожне поле має `<label>` або `aria-label`.
- Модалки — фокус пастка, `Esc` закриває, повернення фокусу на тригер.

## 13. Правила для Claude (як застосовувати)

При створенні / зміні UI-коду:

1. **Спершу перевірити**, чи існує подібний компонент у `src/app` або `src/components`. Не дублювати.
2. **Використовувати тільки токени** з цього документа (CSS-змінні або існуючі класи). Не писати hex-кольорів inline.
3. **Усі відступи — кратні 4.** Якщо пишеш `padding: 15px` — це помилка.
4. **Заголовок має бути 600-weight і `--text-primary`**, опис — 400-weight і `--text-secondary`. Без винятків.
5. **Картки** — білий фон, `12px` радіус, `1px` бордер `--border-subtle`, padding `24px`.
6. **Кнопки** — `8px` радіус, висота `36px`, іконка Lucide 16px.
7. **Іконки — лише `lucide-react`.** Ніяких емодзі, ніяких інших пакетів.
8. **Уникати тіней**, де достатньо бордеру.
9. **i18n:** весь текст — через `t(key)` з `@/i18n/t`. Українська.
10. Якщо запит користувача вимагає порушити правило цього документа — **спитати підтвердження**, а не мовчки відступити від системи.

## 13.1. Розподіл ролей між дизайн-настановами та skill-ами

- **`DESIGN_SYSTEM.md`** визначає, як UI має виглядати в цьому проєкті: кольори, типографіка, spacing, радіуси, іконки, базові компоненти.
- **`emil-design-eng`** відповідає за те, як UI має рухатись і відчуватись: motion, hover / active / focus, easing, duration, micro-feedback, perceived quality.
- **`impeccable`** використовується для ширшого UI/UX critique та final polish: hierarchy, clarity, anti-patterns, design drift, consistency across a full route or flow.
- **`ui-ux-pro-max`** використовується для вибору або генерації сильнішого design direction: palette families, font pairings, product-type patterns, UI styles, and system options for new surfaces.

**Правило:** не змішувати ці ролі. Emil Kowalski не є джерелом повної візуальної системи, а `DESIGN_SYSTEM.md` не замінює interaction polish. Якщо завдання про look-and-feel route цілком, використовувати систему + motion philosophy + critique skill разом.

## 14. Anti-patterns — чого НЕ робити

- ❌ Градієнтні фони карток.
- ❌ Насичені кольори (червоний, синій) як фон великих блоків.
- ❌ `box-shadow` з чорним або >15% alpha.
- ❌ Кілька шрифтів на одній сторінці.
- ❌ `font-size: 11px` і менше (крім службових лейблів, і то рідко).
- ❌ Мікс border-radius (одна картка — 12, її вкладена секція — 6, кнопка — 10).
- ❌ Емодзі замість іконок.
- ❌ `!important` у стилях без критичної причини.
- ❌ Inline hex-кольори — тільки токени.
- ❌ Анімації довші за 300ms для UI-станів.
